# 核心项目动态模态窗 — 需求说明

## 1. 背景与目标

在事业群运营看板（`/dashboard/org/[slug]`）中，「最新核心项目动态」卡片仅展示少量最新条目。业务需要在不离开看板的前提下，**下探查看当前事业群下全部核心项目及其完整动态**，并与看板视觉风格保持一致。

## 2. 范围

| 包含 | 不包含 |
|------|--------|
| 从看板打开/关闭模态窗 | 动态数据的增删改（后台维护） |
| 模态窗内项目列表、筛选、分页、动态折叠展开 | 实时推送、WebSocket |
| 与事业群（slug）绑定的数据范围 | 跨事业群合并查询 |

## 3. 用户场景

1. 用户在某事业群看板页浏览「最新核心项目动态」摘要。
2. 用户点击卡片标题旁的入口（箭头按钮），打开模态窗。
3. 用户在模态窗内按板块、关键词筛选，分页浏览；展开单项目查看全部动态。
4. 用户通过模态窗自带关闭控件（右上角）关闭，回到看板。

## 4. 功能需求

### 4.1 入口与触发

- 入口位于「最新核心项目动态」卡片标题行右侧，与看板其他卡片「查看详情」类操作风格一致（圆形浅底、主题色图标）。
- 悬停有简要说明（如「查看全部核心项目与动态」）。
- 仅对当前路由对应的事业群生效；打开内容须对应该 `slug`。

### 4.2 模态窗容器（外层）

- 使用与设计系统一致的对话框组件（如 Radix/shadcn `Dialog`）。
- **尺寸**：固定宽高为主（当前实现约 880×640px），小屏下不超过视口（`max-h` / `max-w` 约束）。
- **结构**：标题栏与关闭由对话框组件提供；**内容区**为整页 iframe，**不在 iframe 内重复放置关闭按钮**，避免双关闭。
- 模态打开时遵循平台规则：背景滚动由对话框处理；内容区仅中间列表区域可纵向滚动（见内页布局）。
- 关闭方式：仅使用对话框默认关闭（右上角 ×、点击遮罩、Esc 等由组件支持的行为）。

### 4.3 内嵌页（iframe 内）

内嵌静态页路径：`/dashboard/core-project-activity-detail.html?slug={事业群 slug}`。

#### 4.3.1 布局（自上而下）

1. **标题区（固定）**：文案「核心项目动态」。
2. **筛选区（固定）**：
   - **板块归属**：下拉，选项来自接口返回的当前事业群板块列表；含「全部」。
   - **项目号 / 项目名称**：单一搜索框，占位说明支持模糊匹配项目号或名称。
   - **搜索**按钮：点击后应用筛选（见 4.3.3）。
3. **列表区（可滚动）**：展示筛选后的核心项目卡片列表。
4. **分页区（固定）**：页码信息与上一页/下一页。

#### 4.3.2 数据加载

- 内嵌页通过 **同源 API** 拉取数据：`GET /api/org/{slug}/core-project-activity`。
- 需处理：加载中、失败错误提示、无数据提示。

#### 4.3.3 筛选逻辑

- **非实时筛选**：输入过程中不自动过滤；仅点击「搜索」后生效。
- **板块**：与所选板块完全一致；选「全部」则不按板块过滤。
- **关键词**：对「项目号」或「项目名称」做包含匹配（模糊），**二者命中其一即可**（OR）；关键词为空则不过滤。
- 每次有效搜索后，列表回到第一页；展开/折叠状态可按产品约定重置（当前实现为清空展开状态）。

#### 4.3.4 排序

- 项目列表按 **「该项目下最新一条动态的时间」** 倒序排列（越新越靠前）。

#### 4.3.5 分页

- 每页最多 **10** 条**项目**（非动态条数）。
- 显示当前页/总页数及项目总数；首页禁用「上一页」，末页禁用「下一页」。

#### 4.3.6 单项目动态展示

- 每个核心项目展示：项目号、板块、项目名称；其下为动态列表。
- 每条动态展示：状态标签（含子状态若有）、日期；状态类型与看板卡片一致（新增 / 项目信息变更 / 保障活动更新等），颜色区分一致。
- 状态说明：可通过 `title` 等展示与看板一致的说明文案（来自接口 `statusTooltips`）。
- **默认折叠**：每个项目下最多先展示 **3** 条动态（按时间从新到旧）。
- 若动态总数大于 3：提供「展开全部（共 N 条）」；展开后展示全部动态，并可「收起」。

#### 4.3.7 安全与展示

- 服务端或前端对用户可见文本做转义，防止 XSS（内嵌页实现对输出做 HTML 转义）。

### 4.4 看板卡片与数据一致性

- 卡片「最新核心项目动态」仍为跨项目、按动态时间倒序的摘要条数（如 3 条），数据与同一套核心项目数据源一致，避免两套口径冲突。

## 5. 非功能需求

### 5.1 视觉与交互

- 主题色 **#007069**，页面背景 **#f8faff**，简洁商务、无渐变（与平台规范一致）。
- 内嵌页使用 CDN 工具样式（如 Tailwind）实现，**不要求**与主应用共用 React 组件；图表不在此模态内。

### 5.2 无障碍

- 外层对话框具备无障碍标题（如 `DialogTitle` 仅屏阅读器可见）。
- 关闭仅依赖外层对话框控件即可。

### 5.3 技术约束（实现现状）

- 内嵌页为 **独立 HTML**，通过 **iframe** 嵌入；交互与样式在 HTML + 原生 JS 中实现。
- 数据通过 Next.js **Route Handler** 提供 JSON；事业群 slug 校验失败返回 404。

## 6. 数据与接口约定

### 6.1 响应结构（示例）

```json
{
  "sectors": ["板块一", "板块二"],
  "statusTooltips": {
    "新增": "…",
    "项目信息变更": "…",
    "保障活动更新": "…"
  },
  "projects": [
    {
      "id": "项目号",
      "name": "项目名称",
      "sector": "板块归属",
      "activities": [
        { "time": "YYYY-MM-DD", "status": "…", "subStatus": "可选" }
      ]
    }
  ]
}
```

### 6.2 业务扩展

- 后续若改为数据库：保持上述 JSON 结构或兼容映射，内嵌页与 API 路径可逐步替换实现，需求层口径不变。

## 7. 验收要点

- [ ] 各事业群 slug 下打开模态窗，数据范围正确、无串群。
- [ ] 筛选仅在点击「搜索」后生效；关键词同时匹配项目号与名称。
- [ ] 项目排序为「最新动态时间」倒序；分页每页 10 条项目。
- [ ] 单项目动态多于 3 条时出现展开/收起，默认仅显示 3 条。
- [ ] 仅一个关闭入口（对话框关闭），iframe 内无第二关闭按钮。
- [ ] 样式与看板主题色、背景协调；小屏下模态不溢出视口。
